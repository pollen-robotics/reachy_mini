App Autostart

Have a chosen app launch automatically when the robot powers on. Changes take effect on the next boot.

Python module path. The launcher runs /venvs/apps_venv/bin/python -u -m <module>.
Space-separated. No shell quoting.
Autostart service: … · …
Daemon autostart: …
Disable daemon autostart (advanced — out-of-the-box mode)

To return the robot to factory boot behavior (no daemon, no apps), SSH in and run: 

sudo systemctl disable reachy-mini-daemon.service

Re-enable with:

sudo systemctl enable --now reachy-mini-daemon.service

We do not expose this as a UI toggle because disabling the daemon also disables this dashboard.
